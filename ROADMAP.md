# 鲜知贴 Roadmap

鲜知贴当前已经能通过 H5、内置文字助手和 MCP 管理食材。下一阶段重点不只是增加
新的识别方式，而是降低从“买到食材”到“形成可用记录”的完整成本。

本路线图记录产品方向，不代表发布日期。具体顺序会根据原型验证、硬件条件和实际使用
反馈调整。

## 设计原则

- 文字、语音、订单截图和小票照片最终进入同一套食材草稿流程，避免为每种入口维护
  不一致的新增逻辑。
- 自动识别结果先形成可编辑的批量草稿；只追问缺失或低置信度信息，用户确认后再写入。
- 购买日期、保鲜天数、数量和分类尽量根据上下文与用户习惯补全，手动表单始终作为
  可靠的回退入口。
- 音频、图片和偏好数据遵循账号与家庭隔离；用户可以查看、修改或清除系统记住的偏好。
- 不配置模型时，基础食材管理和墨水屏同步仍然可用。

## 计划中的功能

### 1. 墨水屏设备直接语音输入

在硬件上增加麦克风和明确的录音触发方式，让用户可以在冰箱旁直接说出刚买、吃完或
需要修改的食材。语音转写后复用现有助手和批量草稿流程，不让语音识别结果直接、静默
修改库存。

原型阶段需要验证：

- 数字麦克风、录音按键或唤醒方式，以及 ESP32-C3 / ESP32-S3 的资源和引脚条件。
- 录音、上传、转写和失败重试链路，以及弱网下的反馈方式。
- 待机功耗、录音状态提示、误触发和隐私边界。
- 一句话包含多种食材、数量、存放方式和“吃完了”等状态变化时的识别效果。

### 2. 手机订单截图与小票照片识别

在手机 H5 中上传订单截图或拍摄购物小票，自动提取食材名称、规格、数量和可用的订单
日期，排除清洁用品等非食材项目，并生成一次可编辑的批量草稿。

原型阶段需要验证：

- 长截图、多张图片、纸质小票和常见模糊、倾斜场景。
- 相同商品合并、促销信息过滤、规格与实际数量换算。
- 无法从图片确定保鲜期时，使用类别默认值或用户习惯，并明确标记推测字段。
- 图片压缩、上传进度、失败重试，以及原图的保存与删除策略。

### 3. NFC 快速进入管理或对话页面

在冰箱上粘贴低成本 NFC 标签，手机触碰后直接打开鲜知贴的管理页、对话页或专用快速
录入页，减少寻找应用和页面跳转的成本。二维码作为不支持 NFC 时的回退入口。

原型阶段需要验证：

- 登录失效时的返回路径，以及登录完成后恢复到原目标页面。
- NFC 深链接到“管理”“对话”和“快速录入”时的交互取舍。
- 标签只保存公开入口，不写入账号密码、设备令牌或其他敏感信息。
- 公共托管、本地部署和自有域名三种部署方式下的链接配置方式。

### 4. 记住用户偏好和使用习惯

根据用户反复确认和修改的结果，逐步记住家庭常买食材、常用数量单位、默认保鲜天数、
存放方式和录入习惯，减少重复追问。例如支持“和上次一样”“按我平时的牛奶保鲜期”
等表达。

原型阶段需要验证：

- 区分个人偏好、家庭共享偏好和系统通用默认值。
- 只有在有足够重复证据或用户明确要求时才更新偏好，避免一次修改污染长期默认值。
- 在草稿中标明哪些字段来自已记住的习惯，并允许单次覆盖。
- 提供偏好查看、编辑、清除和停止学习的控制入口。

## 建议实施顺序

1. 抽象统一的多来源食材草稿和低置信度字段处理。
2. 完成手机订单截图、小票照片识别与批量确认。
3. 增加 NFC / 二维码快速入口，并观察真实录入频率。
4. 在确认与纠错数据基础上实现可管理的用户偏好。
5. 完成麦克风硬件预研，再决定支持的开发板、录音交互和供电方案。

## 暂不进入近期范围

冰箱内常驻摄像头暂不进入近期路线图。它仍需解决遮挡、光照与冷凝、相似包装、持续
供电、隐私，以及无法可靠区分“移动、吃完或丢弃”等问题。后续可以先尝试手机拍摄冰箱
内部照片，用于周期性盘点和遗漏提醒，再根据效果判断是否值得扩展为固定摄像头。
